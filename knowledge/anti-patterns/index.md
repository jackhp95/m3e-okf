# Anti-patterns

Common mistakes and misuses to avoid, each with the symptom, why it hurts, and the better alternative. Original guidance seeded from Material principles and general UX practice.

## Concepts

| Concept | What it covers |
| --- | --- |
| [Color-only state signaling](/anti-patterns/color-only-state-signaling) | Communicating state, selection, or validity through color alone, with no second cue. |
| [Dialog for non-blocking info](/anti-patterns/dialog-for-non-blocking-info) | Interrupting the user with a modal dialog to deliver information that does not require a decision. |
| [FAB for non-primary actions](/anti-patterns/fab-for-non-primary-actions) | Using the floating action button for anything other than a screen's single most important action. |
| [Over-expressive motion in dense workflows](/anti-patterns/over-expressive-motion-in-dense-workflows) | Applying springy, morphing, attention-grabbing motion to dense, repetitive, task-focused screens. |
| [Too many emphasis levels](/anti-patterns/too-many-emphasis-levels) | Emphasizing so many elements that nothing reads as more important than anything else. |
