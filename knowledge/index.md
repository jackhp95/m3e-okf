# M3 Expressive knowledge bundle

A technology-neutral knowledge bundle for Material 3 Expressive UI design, in Open Knowledge Format (OKF) v0.1. Read this index first, then open only the concept files you need. The tech-specific, CEM-verified component API lives outside this bundle under /implementations/m3e-web.

## Sections

- [Foundations](/foundations/) — Cross-cutting design foundations that apply before any single component: accessibility, layout, adaptive behavior, interaction states, tokens, and content design.
- [Styles](/styles/) — The expressive surface of the system: color, typography, shape, motion, elevation, and icons.
- [Components](/components/) — Neutral per-class concept files — anatomy, usage, and accessibility for each component class in technology-neutral terms. Each links to its tag-level, CEM-verified counterpart under /implementations/m3e-web/components.
- [Expressive](/expressive/) — What Material 3 Expressive (M3E) adds over baseline Material 3: physics-based motion, an expanded shape system with morphing, and an emphasized typography scale — grounded in a large research program. Start with what-changed, then read the concept you need.
- [Patterns](/patterns/) — Recommended, technology-neutral ways to combine components and foundations to solve recurring UI problems — the canonical layouts and flows, with their adaptive behavior.
- [Anti-patterns](/anti-patterns/) — Common mistakes and misuses to avoid, each with the symptom, why it hurts, and the better alternative. Original guidance seeded from Material principles and general UX practice.
- [History](/history/) — How Material Design evolved into Material 3 and Material 3 Expressive, how it rolled out across platforms, and why a web-implementation gap left room for community custom-element libraries like m3e.
