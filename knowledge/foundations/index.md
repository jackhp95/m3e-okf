# Foundations

Cross-cutting design foundations that apply before any single component: accessibility, layout, adaptive behavior, interaction states, tokens, and content design.

## Concepts

| Concept | What it covers |
| --- | --- |
| [Accessibility](/foundations/accessibility) | Names, target sizes, focus, and contrast are construction concerns, not a finishing pass. |
| [Adaptive layouts](/foundations/adaptive-layouts) | A single layout reflows across size classes; canonical patterns give the reflow a known shape. |
| [Content design](/foundations/content-design) | The words in a UI are part of the design — alt text, microcopy, and consistent terminology. |
| [Design tokens](/foundations/design-tokens) | Named variables for visual decisions, flowing through reference, system, and component tiers. |
| [Interaction states](/foundations/interaction-states) | Every interactive element expresses the same set of visual states through a state layer. |
| [Layout](/foundations/layout) | Choose navigation and pane structure by window size class, not by device. |
