# Patterns

Recommended, technology-neutral ways to combine components and foundations to solve recurring UI problems — the canonical layouts and flows, with their adaptive behavior.

## Concepts

| Concept | What it covers |
| --- | --- |
| [Feed](/patterns/feed) | A canonical layout arranging content cards in a grid that reflows from one column to many. |
| [Forms](/patterns/forms) | Structuring input-heavy screens for clarity, validation, and error recovery. |
| [List-detail](/patterns/list-detail) | A canonical layout pairing a scannable list with the selected item's detail, reflowing from stacked to two-pane. |
| [Navigation](/patterns/navigation) | Choosing and adapting the top-level navigation surface — bar, rail, or drawer — across size classes. |
| [Onboarding](/patterns/onboarding) | Introducing a product or feature with a short, skippable, value-first sequence. |
| [Search](/patterns/search) | Presenting search entry, suggestions, and results — from a search bar to a full-screen search view. |
| [Supporting pane](/patterns/supporting-pane) | A canonical layout with a primary content area and a secondary pane holding supporting content. |
